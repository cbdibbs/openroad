# Licensing And Distribution

## Code License

The repository code is licensed under MIT. This applies to gameplay code, pipeline code, validation scripts, and repository documentation unless a file states otherwise.

## Data Separation

Region data is not implicitly covered by the code license. Every distributable region pack carries its own machine-readable attribution and source-license metadata.

## Notices

Third-party notices are tracked in `THIRD_PARTY_NOTICES.md`. Region-specific source usage is tracked inside each region pack's `attribution.json`.

## ODbL Posture

OpenStreetMap-derived route and transportation data is acceptable for this project. The repository is structured so ODbL-related obligations can be satisfied by:

- shipping attribution with region packs
- preserving source versions and lineage in manifests
- keeping data products separate from the application code

## Distribution Rule

The project is intended to remain fully open-source and freely distributable on desktop platforms. No proprietary engine, SDK, or asset dependency should become mandatory for a contributor to build and run the sample region workflow.
